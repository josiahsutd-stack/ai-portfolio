# ADR 0004: Evaluation Over Claims

## Context

A broad AI portfolio can look inflated unless claims are backed by artifacts.

## Decision

Add eval scripts, demo outputs, tests, and claim checks for the strongest projects.

## Consequences

The repo is more inspectable and less marketing-heavy.

## Alternatives Considered

Adding more projects was rejected because breadth was already a weakness.

## Production Change

Production evals would need larger datasets, human review, operational metrics, and external validation.
