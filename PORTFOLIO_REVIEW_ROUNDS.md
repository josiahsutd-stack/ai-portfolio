# Engineering Review History

This change log preserves material credibility defects, the corrections made, and the current artifact that verifies each outcome. It is engineering history, not an external review or endorsement.

| Round | Credibility issue | Repository change | Verifiable outcome |
| --- | --- | --- | --- |
| 1. Claim boundaries | Early project names and summaries implied deeper AI capability than the code supported. | Renamed weak systems, separated selected work from experiments, and added a claims policy plus automated phrase checks. | `scripts/check_claims.py` and `scripts/check_repo_health.py` run in full verification. |
| 2. AEC flagship depth | The original AEC RAG looked like a basic local demo. | Added metadata-rich chunking, four retrieval modes, citations, abstention, evaluation cases, failure analysis, architecture docs, and focused tests. | Synthetic 51-case regression plus public 24-case snapshot over 15 validated downloads. |
| 3. Public-source credibility | Public-document claims lacked fail-closed validation and reproducible identity. | Added source manifests, resolved URLs, PDF/content validation, SHA-256 fingerprints, and snapshot-scoped reporting. | Public `Hit@1 0.952`, `MRR 0.976`, paraphrase `MRR 0.917`, no-answer `1.000`. |
| 4. Embodied AI evidence | Language-to-action logic was initially too close to a scripted grid demo. | Added procedural splits, expert trajectories, random-forest behavior cloning, raw and filtered closed-loop evaluation, interventions, and failures. | The first bounded holdout exposed compounding policy errors and filter dependence. |
| 5. AEC workflow depth | Massing, communication, and QS ideas existed only as portfolio concepts. | Built three bounded systems with typed contracts, focused tests, evaluators, architecture docs, limitations, and generated visuals. | Massing baseline gap, 35-message specification regression, and 21-line QS/tender fixture evaluation. |
| 6. Recruiter hierarchy | Generic text, research, and MLOps exercises diluted the AEC specialization. | Promoted evidence-backed AEC workflow projects and moved generic systems to experiments. Rewrote README, profile, site, role map, and review paths. | Exactly five selected projects and fourteen experiments enforced by repository health checks. |
| 7. Cross-project credibility | A workflow diagram implied continuity without demonstrating that the specification, massing, and QS schemas actually exchanged data. | Added a separate deterministic integration contract with approved-requirement gating, field-level sources, cross-project invariants, schematic geometry provenance, generated traces, and rejection tests. | One labeled synthetic fixture records 5 approved requirements, 16 sourced fields, 96 generated candidates, and 7 priced takeoff lines; tender analysis remains explicitly not run. |
| 8. Embodied representation audit | The embodied project had only an engineered classical policy, while future visual-policy language could be mistaken for implemented capability. | Added a fully observable semantic-raster encoder, a deterministic MLP baseline on the same demonstrations and holdout, model-specific tests and cards, a generated comparison visual, and explicit non-perception boundaries. | Across 96 holdout scenarios, the random forest reaches `0.855` action accuracy and `0.698` filtered success; the raster MLP reaches `0.478` and `0.292`. The weaker neural result remains published. |
| 9. Egocentric observation and terminal-semantics audit | The world-frame raster showed that a neural label alone added little, but the project did not test observation alignment. Visual QA also exposed a timeout path that could misclassify nonterminal battery recovery as task success. | Added a 5x5 agent-centered encoder and third shared-split MLP; then restricted success to task-specific terminal events, penalized unnecessary charging, and added recovery/timeout regression tests before regenerating all evidence. | Egocentric action accuracy is `0.834`; raw success is `0.573`; corrected filtered success is `0.760` with `943` interventions. The discarded inflated result is not used on public surfaces. |
| 10. Evidence-presentation audit | A candidate-authored scorecard used numeric ratings and interview language that could be mistaken for an independent assessment. The claim checker also skipped static-site HTML and integration docs. | Replaced the scorecard with a source-linked coverage matrix, added explicit presentation rules, expanded scanning to Markdown and HTML, and added regression tests plus an obsolete-file invariant. | `scripts/check_claims.py` now audits 163 public text files; the evidence audit reports implemented scope and missing validation without self-ratings. |
| 11. Flagship service-boundary audit | The AEC project exposed FastAPI endpoints and local logs without authentication, readiness semantics, request tracing, service-level tests, or a reproducible contract artifact. Deployment recipes could be read as stronger evidence than the repository contained. | Added a fail-closed service factory, bounded request IDs, public liveness/readiness, default log redaction, process-local metrics, focused migration/error tests, and a deterministic in-process evaluator. Reframed hosting instructions as unverified options. | The checked-in synthetic ASGI run passes 12/12 contract checks and records 9 requests before the metrics response; no external deployment or traffic is claimed. |
| 12. Embodied visual-input audit | The embodied project compared state encodings but had no pixel-derived policy, so its visual and VLA boundaries remained entirely hypothetical. | Added a deterministic egocentric RGB renderer, appearance-augmented MLP training, a held-out work-light palette, raw/filtered rollouts, pixel artifacts, a model card, and schema/shift tests. | Standard RGB accuracy/filter success are `0.813`/`0.719`; unseen-palette results fall to `0.417`/`0.427`, with raw success `0.000` and 3,315 interventions. Pixels remain explicitly state-rendered, not camera evidence. |
| 13. Flagship reliability-evidence audit | The flagship exposed only process-local metrics, so restart durability and latency/error objectives were named as future work rather than exercised behavior. | Added bounded payload-free SQLite request telemetry, explicit insufficient/pass/fail query objectives, fail-open telemetry errors, a concurrent in-process workload, app reconstruction, focused tests, and generated evidence. | The fixed synthetic workload passes 14/14 checks: 48/48 responses return 200, zero server errors occur, P95 is at or below 500 ms, and all 48 query rows remain after app reconstruction. This is not network, sustained-load, uptime, or deployment evidence. |
| 14. Embodied command-boundary audit | Pixel-derived observation evidence still ended at a discrete grid transition, while the documented absence of physics left collision behavior entirely inferred from hand-authored rules. | Added a pinned MuJoCo dependency, continuous planar command adapter, bounded actuators, named rigid-contact telemetry, balanced learned-policy holdout subset, compact evidence artifacts, generated comparison visual, and geometry/contact/determinism tests. | Raw egocentric commands contact named rigid geometry on 51/150 movements; filtered egocentric and A* traces record 0 contacts across 148 and 98 movements. The evidence is explicitly limited to planar command replay, not a mobile-robot controller, ROS, hardware, or physical safety. |
| 15. Flagship statistical-uncertainty audit | Public retrieval headlines reported point estimates from 24 authored cases, and the sorted ablation table could be read as stronger mode-selection evidence than the sample supported. | Added deterministic Wilson and bootstrap intervals, paired mode comparisons over matching case IDs, mismatch/tie/small-sample tests, generated JSON/report/SVG artifacts, and claim-checked recruiter-facing caveats. | Public Hit@1 is `0.952 [0.773, 0.992]` over 21 answerable cases; no-answer is 2/2 with interval `[0.342, 1.000]`; hybrid-vs-BM25 MRR delta is `0.012 [0.000, 0.036]` and therefore inconclusive on this authored set. |
| 16. Retrieval-granularity audit | Document-level Hit@1 could be read as evidence that the supporting passage itself ranked first, while the public set had no auditable exact-chunk/page targets. | Added 21 candidate-authored exact-chunk targets, 18 page targets, corpus validation, unique repeated-heading chunk IDs, granular metrics, uncertainty intervals, per-case label audit artifacts, focused tests, and a generated comparison visual. | Document Hit@1 remains `0.952`; exact-target Hit@1 is `0.810` and MRR is `0.881`; page-target Hit@1 is `0.778`. The labels are explicitly not independent expert annotations. |

The audit intentionally preserves limitations and negative evidence. Perfect fixture scores are labeled as deterministic regression results, not general model-quality claims.
