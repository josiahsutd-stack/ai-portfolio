# AEC Design-To-Cost Integration Contract

This integration executes a narrow handoff across three selected projects:

1. role-tagged messages become a requirement ledger;
2. three explicitly approved requirements enter a sourced massing scenario;
3. a feasible, storey-matched massing option becomes a one-storey schematic envelope;
4. the envelope is measured and priced with a bundled synthetic rate library.

The integration is deterministic and local. It is evidence that the project interfaces exchange traceable data and reject invalid handoffs. It is not a claim of automated architectural design, code compliance, professional cost planning, or tender evaluation.

![Executed synthetic workflow trace](demo_outputs/workflow_trace.svg)

## Run

From the repository root:

```bash
python integrations/aec-design-to-cost/run_workflow.py
python -m pytest tests/test_aec_workflow_integration.py
```

No API key, hosted model, or private data is required.

## Scope Decisions

- Only approved site area, GFA, and storey count are mapped automatically.
- Budget and accessibility remain in the trace but outside geometric and costing automation.
- Site dimensions, setbacks, access points, limits, and objective weights are explicit synthetic site inputs; no code or site facts are inferred.
- The QS input is a one-storey rectangular envelope derived from mass footprints, not a developed floor plan.
- The synthetic cost build-up is not compared with the approved project budget because their scopes differ.
- Tender analysis is not invoked because no tender submission belongs to the derived plan.
